Container

The container function aims to set the first-level horizontal constraint. This translates into a maximum width (generaly 960px) that the content must not exceed. Because this is the first-level contraint, this function should be used on first-level elements (i.e. : <main>, <header>, <footer>). 

container(1080px, 32px, 800px, 32px) //used on <main>
container(none, 0, 800px, 64px) //used on <footer>

The code above is used on this demo page. As you can see, the function is used with 4 parameters.

The first parameter aims to define the max width of the container.

The second parameter defines the minimum margin on each side of the container. Above a specific window width, the margin is set to the given number.

The third parameter defines the maximum "inner width". By inner, I mean the width remaing after the padding been applied.

Similar to the second parameter, the fourth parameter defines a minimum padding.

Extra parameters :

The fifth parameter defines the alignment of the "outer container" relatively to the window.

The sixth parameter defines the alignment of the "inner container" relatively to the "outer container".

Columns

The columns function allows to define complexe columns pattern. In contrary to bootstrap's grid you are not limited to 12 columns. Most of the concept comes from NeutronCSS built on SASS, with some additions :

  • columns width
  • order and alignment
  • blank/empty columns, similar to the offset in bootstrap
  • margin and padding patterns

Summary

Here is how to read the code following :

columns(7 right, "div", 12px space-between, 12px equal, 1 -1 1 3, 2 3 1)

Grid made of 7 columns aligned to the right.

Targeted tag is div.

There is a 12px space between each columns.

The content is padded by 12px with the equal pattern.

The columns pattern is : one column of weight 1, one blank column of weight 1, another column of weight 1 and a last columns of weight 3.

The column order from left to right is : second, third then first. This pattern is repeted every 3 div.

Boxes

The concept of boxes stand on css flex-box. In contrary to columns, boxes width and position adapts depending on their content and their number. Therefore, if these two parameters are not known in advance, the boxes() function may be more suitable than columns().

Horizontal properties

You can control the alignment using the following arguments: top, center, bottom, stretch space-around and space-between.

You can also reverse the wrapping direction using reverse.

+boxes(stretch reverse, single center){
    boxes-margin(12px, space-around);
}

Vertical properties

By default, boxes wraps on multiple lines. Like for the horizontal axis, you can reverse the wrapping direction using reverse. You can disable wrapping using single.

Like for the horizontal axis, you can control the alignment using the following arguments: top, center and bottom.
When wrapping is enabled : space-around, space-between and stretch.

height: 300px;
boxes(space-between, space-around reverse, "div", 40%)

Note:

By default, boxes are stretched on their line.
Using single, because there is only one line, you control the boxes alignment on the line (overwriting the stretching behavior).
But when line wrap is enabled, you control the lines alignment in the container. Therefore boxes will have the same size on their line.

Using the boxes-margin() function is only revelant on single line mode, with horizontal shrink enabled.
